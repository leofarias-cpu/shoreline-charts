Switch is a component for customizing configurations. It represents a physical switch that allows users to turn things on or off, providing control over actions and preferences. Tapping a toggle switch is a two-step action: selection and execution. It should reflect an action that triggers a state, never both at once.

## Example

```jsx live
function Example() {
  const [checked, setChecked] = React.useState(false)

  return (
    <Switch
      label="Label"
      ariaLabel="label"
      state={{
        checked,
        onChange: () => setChecked(!checked),
      }}
    />
  )
}
```

## Variations

### Standalone

When using a standalone `Switch` you should provide an `aria-label` property value. As a `form` component, it should have a label specified to guarantee accessibility.

```jsx live
function Example() {
  const [checked, setChecked] = React.useState(false)

  return (
    <Switch
      label="Label"
      aria-label="label"
      checked={checked}
      onChange={() => setChecked(!checked)}
    />
  )
}
```

### Checked States

There are two checked states: `not checked`, and `checked`.

```jsx live
function Example() {
  return (
    <>
      <Set>
        <Switch aria-label="label-1" label="Label" />
        <Switch aria-label="label-disabled-1" disabled label="Label" />
      </Set>
      <Set>
        <Switch aria-label="label-2" checked label="Label" />
        <Switch aria-label="label-disabled-2" checked disabled label="Label" />
      </Set>
    </>
  )
}
```

### Multiple Switches

Oftentimes we need to render multiple switches and store the checked values in an array. It can be easily done using our `useSwitchState` hook, you just need to pass the hook return object to the switches `state` property and define a `value` for each `Switch`.

```jsx live
function Example() {
  const switchState = useSwitchState({ state: [] })

  return (
    <>
      <Text>Selected switches: {switchState.state.join(', ')}</Text>
      <Switch state={switchState} value="switch1" label="Label" />
      <Switch state={switchState} value="switch2" label="Label" />
      <Switch state={switchState} value="switch3" label="Label" />
    </>
  )
}
```

## State

There are two ways of handling the state in the `Switch`.

### checked and onChange

You can use the properties `checked` and `onChange` to handle if the Switch is checked and its values have changed.

```jsx live
function Example() {
  const [checked, setChecked] = React.useState(false)

  return (
    <Switch
      checked={checked}
      onChange={() => setChecked(!checked)}
      label="Label"
    />
  )
}
```

### useSwitchState

For convenience, we also provide a hook that already implements the state logic for you. It can be very handy if you have a group of switches and want to handle the states of each one. You should pass the hook return to the `state` property and also define a `value` for each `Switch`.

```jsx live
function Example() {
  const switchState = useSwitchState({ state: false })

  return <Switch state={switchState} label="Label" />
}
```
