Switch triggers allow merchants to instantly turn on or off the state of a feature or item.

When using a standalone `Switch` you should provide an `aria-label` property value. As a `form` component, it should have a label specified to guarantee accessibility.

function Example() {
  const [checked, setChecked] = React.useState(false)

  return (
    <Switch
      label="Label"
      aria-label="label"
      checked={checked}
      onChange={() => setChecked(!checked)}
    />
  )
}

It is sometimes necessary to render multiple switches and store the checked values in an array. It can be done using our `useSwitchState` hook. You just need to pass the hook return object to the switches `state` property and define a `value` for each `Switch`.

<br/>

```jsx live
function Example() {
  const switchState = useSwitchState({ state: [] })

  return (
    <>
      <Text>Selected switches: {switchState.state.join(', ')}</Text>
      <Switch state={switchState} value="switch1" label="Label" />
      <Switch state={switchState} value="switch2" label="Label" />
      <Switch state={switchState} value="switch3" label="Label" />
    </>
  )
}
```

For convenience, we also provide a hook that already implements the state logic for you. It can be very handy if you have a group of switches and want to handle the states of each one. You should pass the hook return to the state property and also define a value for each Switch.

<br/>

```jsx live
function Example() {
  const switchState = useSwitchState({ state: false })

  return <Switch state={switchState} label="Label" />
}
```

<details>
  <summary><h4>What should be the label of a Switch?</h4></summary>
  <div>
    <ul>
        <li>Start the label with a verb in the infinitive. For example, use <em>Distribute products in marketplaces</em> instead of <em>Products should be distributed in marketplaces</em>.</li>
        <li>Write the label so that it corresponds to the default state of the Switch.</li>
        <li>Use sentence case but capitalize proper nouns.</li>
    </ul>
  </div>
  <br />
  <div>
    <strong>What to avoid:</strong>
    <ul>
      <li>Don't use generic verbs. For example, write <em>Distribute products in marketplaces</em> or <em>Sell products from sellers</em> instead of <em>Enable marketplace</em> or <em>Enable Sellers.</em></li>
      <li>Don't leave entities implicit. For example, write <em>Suggest pages for search engine results</em> instead of <em>Suggest for results</em>.</li>
      <li>Don't use personal pronouns. For example, write <em>Send to personal email</em> instead of <em>Send to my email</em>.</li>
      <li>Don't write long labels. Use the help text to provide additional context.</li>
      <li>Don't write labels that vary a lot in length between options</li>
      <li>Don't write more than one sentence.</li>
      <li>Don't use periods or commas.</li>
    </ul>
  </div>
</details>

<details>
  <summary><h4>What should be the help text of a Switch?</h4></summary>
  <div>
    <ul>
      <li>Include a help text only for critical choices where the consequences of switching the state need to be explained in more detail.</li>
      <li>Write the help text as if you are completing the following sentence: <em>If you switch the default state, you or the system will [help text]</em>.</li>
      <li>Start with verbs in the simple present tense. For example, for the label <em>Distribute products in marketplaces</em>, the help text should be <em>Sends active products to connected marketplaces</em> and not <em>Send active products to connected marketplaces</em>.</li>
      <li>Use sentence case but capitalize proper nouns.</li>
    </ul>
  </div>
  <br />
  <div>
    <strong>What to avoid:</strong>
      <ul>
        <li>Avoid including links. Use an <a href="https://admin-ui.vercel.app/components/alert">Alert</a> component if this is necessary.</li>
        <li>Avoid writing more than one sentence.</li>
        <li>Avoid using periods or commas.</li>
      </ul>
  </div>
</details>
