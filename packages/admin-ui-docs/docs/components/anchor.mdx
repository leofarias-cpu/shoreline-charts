You can use the `className` property to customize any style.

  Link
</Anchor>

<details>
  <summary>
    <h4>When to use a Button, a Tab or a Menu instead of an Anchor?</h4>
  </summary>
  <ul>
    <li>Choose a Button if the action is not positioned in a text, requires higher visual prominence, or triggers a process such as submitting a form. Only choose a Button for a navigation action when it is in the header of a Card, Modal or Page.</li>
    <li>Choose a Tab if the the purpose of the action is to navigate within a container and the different views are mutually exclusive, meaning that only one view can be visible at a time.</li>
    <li>Choose a Menu if the action is rarely used or is repeated for each item in a list, even when the purpose of the action is to simply navigate.</li>
  </ul>
</details>

<details>
  <summary>
    <h4>What should be the action of an Anchor?</h4>
  </summary>
  Anchors are usually for navigation to other pages, but depending on the context, they can be used for other purposes. All the technically supported possibilities are listed below:
  <ul>
    <li>Navigate to another page.</li>
    <li>Navigate to a specific section of another page.</li>
    <li>Navigate to highlighted text on another page.</li>
    <li>Navigate to another section of the current page.</li>
    <li>Open the default email client with pre-filled content.</li>
    <li>Dial a number in the default telephone app.</li>
  </ul>
  <br/><strong>What to avoid:</strong><br/><br/>
  <ul>
    <li>Don't use an Anchor to download a file, prefer using a <a href="https://admin-ui.vercel.app/components/button">Button</a>.</li>
    <li>Don't use an Anchor to expand or collapse content, prefer using a <a href="https://admin-ui.vercel.app/components/button">Button</a>.</li>
  </ul>
</details>

<details>
  <summary>
    <h4>When should an Anchor open in the same tab or in another tab or window?</h4>
  </summary>
  <ul>
    <li>When in doubt, avoid opening an Anchor in a new tab. Since there are keyboard shortcuts to achieve this behavior, setting this behavior as default takes away the merchantâ€™s choice.</li>
    <li>Open an Anchor in another tab when it leads outside the VTEX Admin, such as to documentation or to a partnerâ€™s website. Also, open in a new tab to avoid interrupting a process in the current tab, such as filling in a form or watching a video.</li>
  </ul>
</details>

      <li>Anchors use by default the <code>$body</code> token with a predefined weight and size. An underline appears only on hover, and color tokens for tertiary actions are used.</li>
    </ul>
    <br/><strong>What to avoid:</strong><br/><br/>
    <ul>
      <li>Donâ€™t position an Anchor in the header or footer of a container, prefer to use a <a href="https://admin-ui.vercel.app/components/button">Button</a> instead.</li>
    </ul>
</details>

<details>
  <summary>
    <h4>What should be the behavior of an Anchor after it is clicked?</h4>
  </summary>
  <ul>
    <li>When merchants might find it useful to know if they have clicked on a link before, such as when searching through a list of links, change the Anchor color to indicate that it has been visited.</li>
    <li>Once an Anchor is visited, it is registered in the browser history and continues to display the visited state until this history entry is cleared.</li>
  </ul>
  <br/><strong>What to avoid:</strong><br/><br/>
  <ul>
    <li>Don't customize the color of a visited Anchor, maintain the <code>$fg.action.tertiaryPressed</code> token to ensure that merchants can easily recognize visited links.</li>
  </ul>
</details>

  <summary>
    <h4>What should be the text of an Anchor?</h4>
  </summary>
  <ul>
    <li>If the link is within a paragraph, the capitalization should not be modified, it should not start with a verb and the words should define the content that will be visited.</li>
    <li>If the link is at the end of a paragraph, it should be in sentence case and start with a verb, such as <em>Learn more</em> or <em>View details</em>.</li>
  </ul>
  <br/><strong>What to avoid:</strong><br/><br/>
  <ul>
    <li>Donâ€™t use more than three words in an Anchor.</li>
    <li>Donâ€™t use the verb <em>See</em>. Prefer using <em>View</em></li>
