```jsx live
function Example() {
  function useTreeState({ values }) {
    const { value: group, setValue: setGroup } = useCheckboxState({
      initialValue: [],
    })
    const { value: items, setValue: setItems } = useCheckboxState({
      initialValue: [],
    })

    // updates items when group is toggled
    React.useEffect(() => {
      if (group === true) {
        setItems(values)
      } else if (group === false) {
        setItems([])
      }
    }, [group, setItems, values])

    // updates group when items is toggled
    React.useEffect(() => {
      if (items instanceof Array && items.length === values.length) {
        setGroup(true)
      } else if (items instanceof Array && items.length) {
        setGroup('indeterminate')
      } else {
        setGroup(false)
      }
    }, [items, setGroup, values])

    return { group, items, setItems, setGroup }
  }

  const values = React.useMemo(() => ['Apple', 'Orange', 'Watermelon'], [])
  const { group, setGroup, items, setItems } = useTreeState({ values })

  return (
    <CheckboxGroup label="Fruits" id="fruits-checkbox-group" direction="column">
      <Checkbox
        label="Select All (Group Control)"
        state={{ value: group, setValue: setGroup }}
      />
      {values.map((fruit, key) => (
        <Checkbox
          state={{ value: items, setValue: setItems }}
          value={fruit}
          label={fruit}
        />
      ))}
    </CheckboxGroup>
  )
}
```

<details><summary>When to use the Switch, Radio, or Combobox Multiple components instead of the Checkbox component?</summary>
<p>
  <ul>
    <li>Choose the Switch component if the action is instant and does not require confirmation. For example, use a switch in a product form to show a new form section only if the product has variants.</li>
    <li>Choose the Radio component if only one option can be selected at a time and there are up to five options.</li>
    <li>Choose the Combobox Multiple component if multiple options can be selected and there are more than five available options.</li>
  </ul>
</p>
</details>

<details><summary>When to use a Checkbox Group instead of a Checkbox?</summary>
<p>Since the Checkbox label should only correspond to its value, the choice should be contextualized using the label of a Checkbox Group or the title of a form section. This is why the Checkbox Group is usually the right choice even for binary choices where there is only a single Checkbox.</p>
</details>

<details><summary>When to use a vertical or a horizontal Checkbox Group?</summary>
<p>It is more scalable to distribute options in a vertical direction. Only distribute horizontally if â€” considering localization â€” all their labels fit in a single line, have up to 20 characters and donâ€™t vary a lot in length.</p>
</details>

<details><summary>When should a Checkbox be disabled?</summary>
<p>
  <ul>
    <li>Use a disabled Checkbox only if in some condition it becomes enabled. Example: when an option is incompatible with another option in a form.</li>
    <li>Use a <a href="https://admin-ui.vercel.app/components/tooltip">tooltip</a> trigger to explain why the Checkbox is disabled. Display the information when hovering over the tooltip trigger â€” not the Checkbox itself.</li>
  </ul>
</p>
</details>

<details><summary>When should the indeterminate state be used?</summary>
<p>
In scenarios where it might be necessary to check or uncheck all options, a Checkbox can be placed in the beginning of the group with the <em>Select all</em> label. Make sure options below are nested and vertically aligned with the <em>Select all</em> label.<br /><br />
The indeterminate state should be displayed in these scenarios when only a few options (not all) are checked. When clicking the indeterminate state, all options must be checked. When clicking for a second time, all options must be unchecked.
</p>
</details>

<details><summary>What should be the default value of a Checkbox?</summary>
<p>
  <ul>
<li>Preselect an option when this can make the merchant more efficient and when it isnâ€™t important for the choice to be conscious.</li>
<li>Common or recommended options can be preselected. Information that the merchant has previously provided can be preselected to indicate the option that should be chosen.</li>
<li>When a recommended option is preselected, add <em>(Recommended)</em> to the end of its label so it can still be identified after edits.</li>
  </ul>
</p>
</details>

<details><summary>What should be the label of a Checkbox?</summary>
<p>
  <ul>
    <li>Write the label so it corresponds to the value of the Checkbox.</li>
    <li>Use sentence case but capitalize proper nouns.</li>
  </ul>
  <br /><br />
  What to avoid:
  <ul>
    <li>Don't write long Checkbox labels. Use the help text or the Checkbox Group label to provide additional context when necessary.</li>
    <li>Don't write Checkbox labels that vary a lot in length between options.</li>
    <li>Don't use personal pronouns. For example, write <em>Receive promotional emails</em> instead of <em>I want to receive emails</em>.</li>
    <li>Don't include redundant words. For example, use only <em>S</em>, <em>M</em>, and <em>L </em>instead of <em>Size S</em>, <em>Size M</em>, and <em>Size L</em>.</li>
    <li>Don't include negative words. For example, write <em>Receive promotional emails</em> instead of <em>I don't want to receive emails</em>.</li>
  </ul>
</p>
</details>

<details><summary>What should be the label of a Checkbox Group?</summary>
<p>
  <ul>
    <li>Include the name of the entity that is being listed. For example, use the label <em>Fruits </em>when the options are <em>Apple</em>, <em>Orange </em>and <em>Banana</em>.</li>
    <li>Visually hide the label of the Checkbox Group when the form section title already contains the name of the entity that is being listed.</li>
    <li>Use sentence case, but capitalize proper nouns.</li>
  </ul>
  <br /><br />
  What to avoid:
  <ul>
    <li>Don't include redundant words. For example, don't use <em>Product characteristics</em> as the label for a Checkbox Group inside a form page that includes <em>Product</em> in its title â€“ use only <em>Characteristics</em>.</li>
    <li>Don't use verbs. For example, instead of <em>Select characteristics</em>, use only <em>Characteristics</em>.</li>
    <li>Don't use interrogations.</li>
  </ul>
</p>
</details>

<details><summary>What should be the help text of a Checkbox?</summary>
<p>
  <ul>
    <li>Include a help text only for critical choices where the consequences of checking the option need to be explained in more detail.</li>
    <li>Write the help text as if you are completing the following sentence: <em>If you check this option, you or the system will [help text]</em>.</li>
    <li>Start with verbs in the simple present tense. For example, for the value <em>Fragile</em> the help text should be<em> Prints this information on the shipping label</em> and not <em>Print this information on the shipping label</em>.</li>
    <li>Use sentence case, but capitalize proper nouns.</li>
  </ul>
  <br /><br />
  What to avoid:
  <ul>
    <li>Don't include links. Use an alert component if this is necessary.</li>
    <li>Don't write more than one sentence.</li>
    <li>Don't use periods or commas.</li>
  </ul>
</p>
</details>

<details><summary>What should be the error text of a Checkbox or a Checkbox Group?</summary>
<p>
  <ul>
    <li>Prevent errors whenever possible. When an option is checked, disable other incompatible fields or options automatically.</li>
    <li>In a Checkbox Group, if at least one option needs to be checked and none is checked by the merchant, the error text should be <em>Check at least one option</em>.</li>
    <li>Use the imperative form. For example, write <em>Check at least one option</em> instead of <em>At least one option needs to be checked</em>.</li>
  </ul>
  <br /><br />
  What to avoid:
  <ul>
    <li>Don't use periods or commas.</li>
    <li>Don't use incomplete sentences. For example, write <em>Check at least one option</em> instead of <em>Check at least one</em>.</li>
  </ul>
</p>
</details>
