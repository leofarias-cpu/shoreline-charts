## What is this?

Pagination triggers allow merchants to view the size of a list and navigate between pages.

{' '}

<br />

You can persist the pagination state within the browser's query params using the `useQueryPaginationState` hook, instead of `usePaginationState`. Both hooks are used in the same way. The application should be wrapped by `QueryStateProvider`.

{' '}

<br />
Watch how your url changes. <br /> <br />

```jsx live
function Example() {
  const Content = () => {
    const state = useQueryPaginationState({ pageSize: 5, total: 35 })

    return <Pagination state={state} />
  }

  return (
    <QueryStateProvider>
      <Flex justify="end">
        <Content />
      </Flex>
    </QueryStateProvider>
  )
}
```

</PageSection>

```jsx live
function Example() {
  const state = usePaginationState({ pageSize: 5, total: 35 })

  const { paginate } = state

  return (
    <Flex justify="space-between">
      <Inline>
        <Button onClick={() => paginate({ type: 'navigate', page: 3 })}>
          Navigate
        </Button>
        <Button onClick={() => paginate({ type: 'prev' })}>Previous</Button>
        <Button onClick={() => paginate({ type: 'next' })}>Next</Button>
        <Button onClick={() => paginate({ type: 'reset' })}>Reset</Button>
        <Button onClick={() => paginate({ type: 'setTotal', total: 100 })}>
          Set total
        </Button>
      </Inline>
      <Pagination state={state} />
    </Flex>
  )
}
```

{' '}

<Admonition type="caution">
  <p>
    Be careful when using the <code>setTotal</code> action, because if you set
    the total to a smaller value it may lead your current page to an invalid
    one.
  </p>
</Admonition>

      <li>Pagination should always be in the top right of a container.</li>

      </li>

        component to correctly position the Pagination.
      </li>
    </ul>
  </div>

<PageSection

>

<details>
  <summary>
    <h4>How many items should be shown per page of a list?</h4>
  </summary>
  <div>
    <ul>
      <li>The standard is to show 25 items per page of a list.</li>
      <li>
        When the list is inside a container where vertical scroll should be
        avoided, such as a Modal or a Card, limit the amount of items
        accordingly.
      </li>
    </ul>
  </div>
  <br />
  <div>
    <strong>What to avoid:</strong>
    <ul>
      <li>
        Donâ€™t allow merchants to customize the number of results per page.
      </li>
    </ul>
  </div>
</details>

<details>
  <summary>
    <h4>When to show a loading Pagination?</h4>
  </summary>
  <div>
    When the list is loading, since the number of results is not loaded and the
    user canâ€™t navigate between pages yet.
  </div>
</details>

<details>
  <summary>
    <h4>When should the number of results be updated?</h4>
  </summary>
  <div>
    <ul>
      <li>
        When the merchant edits the filter values or the search query,
        the total number of results should inform the current state of the list.
      </li>
      <li>
        Only load new items when the merchant explicitly triggers an action to
        update the results, such as the <em>Refresh</em> button in the web
        browser.
      </li>
      <li>
        When the user has just created an item and returns to the listing page,
        they should go to the first page with updated results.
      </li>
      <li>
        When the user has just edited an item and returns to the listing page,
        they should see a list with the same items as before, but with the
        edited item updated.
      </li>
    </ul>
  </div>
  <br />
  <div>
    <strong>What to avoid:</strong>
    <ul>
      <li>
        Donâ€™t load new items while the merchant is navigating between pages.
      </li>
    </ul>
  </div>
</details>

<details>
  <summary>
    <h4>When should the current state of a list be stored in the URL?</h4>
  </summary>
  <div>
    The complete search query, which includes the filters, the current page and
    the search terms, should be reflected in the URL. This allows the merchant
    to easily return to the customized view and easily share it.
  </div>
  <br />
  <div>
    <strong>What to avoid:</strong>
    <ul>
      <li>Donâ€™t use local storage or cookies to store the query.</li>
    </ul>
  </div>
</details>
