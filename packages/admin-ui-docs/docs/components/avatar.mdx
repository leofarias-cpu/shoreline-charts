  ## What is this?

  Avatar indicators visually represent a person with an Admin user account.

  </IntroSection> <br/>

  
## Examples

### Palette

The `Avatar` can be rendered in multiple palettes, this means that the `color` and `background-color` properties change according to its `palette` property value. It has `lightBlue` value by default.

```jsx live
<Stack direction="row">
  <Avatar label="lightBlue" />
  <Avatar label="green" palette="green" />
  <Avatar label="orange" palette="orange" />
  <Avatar label="cyan" palette="cyan" />
  <Avatar label="purple" palette="purple" />
  <Avatar label="teal" palette="teal" />
  <Avatar label="grey" palette="grey" />
</Stack>
```

```jsx
import { Avatar } from '@vtex/admin-ui'

function Example() {
  return <Avatar label="Label" />
```

<details>
   <summary>
      <h4>What should be the color of an Avatar?</h4>
   </summary>
   Ideally, a color would be generated for each user to identify them consistently throughout the Admin. However, since this color information is still not available for each Admin user, it is recommended to use the default color (<code>lightBlue</code>) to avoid inconsistencies between pages and incorrect associations.
</details>

<details>
   <summary>
      <h4>What should be the position and size of an Avatar in a container?</h4>
   </summary>
   <ul>
      <li>In a table listing individuals, such as a list of registered users, the Avatar can be positioned as a column just before the name in each row, with a size of 44px.</li>
      <li>In a table with content created by the merchant, such as products and promotions, the Avatar can be one of the last columns, identifying who made changes, with a size of 24px.</li>
      <li>In a comments section, the Avatar should be positioned to the left of the comment text, with a size of 44px.</li>
   </ul>
</details>

  </PageSection>

  <PageSection
    id={tabToc.designReference[3].id}

<details>
   <summary>
      <h4>What should happen with an Avatar on click or hover?</h4>
   </summary>
   <ul>
      <li>On click, nothing should happen unless the Avatar is within an element that already has a click behavior, such as a table row.</li>
      <li>On hover, when the full name of the Admin user is not already displayed next to the Avatar, a Tooltip should appear with this information.</li>
   </ul>
</details>

  >

<details>
   <summary>
      <h4>What should be the initials or the photo of an Avatar?</h4>
   </summary>
   <ul>
      <li>Ideally, the initials should be shown only when the user does not have a registered photo. However, this component doesn't support images since Admin users still can't add a photo to their profiles.</li>
      <li>We recommend always using the uppercase initials for the Avatar.</li>
      <li>When the Avatar size is 24px, use the first letter of the user's name.</li>
      <li>When the Avatar size is 44px, use the first letter of the user's name and the first letter of their last surname.</li>
      <li>Consult the Localization team to understand how Avatar indicators should work in other alphabets or character systems.</li>
   </ul>
</details>

<details>
   <summary>
      <h4>When to use an Avatar together with the full name?</h4>
   </summary>
   <ul>
      <li>Always use the Avatar together with the person's full name. Depending on the available space, this text can also appear on hover in a Tooltip.</li>
      <li>In a comments section, include the full name always visible next to the Avatar.</li>
      <li>In a Table that includes the last modification made, an Avatar for the person who made the change can be in the same column as the modification date or in a separate column, depending on the space available. If they are in the same column, keep the date visible and show a tooltip with the full name on hover.</li>
   </ul>
   <br/><strong>What to avoid:</strong><br/><br/>
   <ul>
      <li>Don't change the capitalization of the user's full name. Prefer to display the name as was registered by the user in their profile.</li>
      <li>Don't truncate one or more surnames. Prefer to include the full name and wrap the text if necessary.</li>
   </ul>
</details>
