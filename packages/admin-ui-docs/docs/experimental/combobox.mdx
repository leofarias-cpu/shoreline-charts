### Custom rendering
<br />

You can use objects instead of strings for your Combobox's list and define a custom rendering function for each option. For this you must also define a `getOptionValue` function (for converting the object shape into a unique string value),
a `renderOption` function which should return a ReactNode that represents this object visually. If you don't define these 2 functions and you use objects on your list you will get an unexpected behavior.<br /><br />

The value obtained by the `getOptionValue` function will be the one considered when performing the search based on the user input.<br /><br />

### Async
<br />

You can use the `deferredValue` and `setMatches` to handle searches in async environments.<br /><br />

```jsx noInline live
const ComboboxField = experimental_ComboboxField
const ComboboxPopover = experimental_ComboboxPopover
const useComboboxState = experimental_useComboboxState

// just a sample API
function api(search) {
  const items = [
    { value: 'Brazil' },
    { value: 'Bahamas' },
    { value: 'Belarus' },
    { value: 'France' },
    { value: 'Ukraine' },
    { value: 'Australia' },
    { value: 'Afghanistan' },
    { value: 'Albania' },
    { value: 'Algeria' },
    { value: 'American Samoa' },
    { value: 'Andorra' },
    { value: 'Angola' },
    { value: 'Anguilla' },
    { value: 'Antarctica' },
    { value: 'Antigua and Barbuda' },
    { value: 'Argentina' },
    { value: 'Armenia' },
    { value: 'Aruba' },
    { value: 'Austria' },
    { value: 'Azerbaijan' },
  ]

  const res = items.filter((item) =>
    item.value.toLowerCase().startsWith(search.toLowerCase())
  )

  return new Promise((resolve) => setTimeout(resolve, 1000, res))
}

function Example() {
  const combobox = useComboboxState()

  useEffect(() => {
    if (combobox.deferredValue === '') {
      combobox.setMatches([])
    } else {
      combobox.setLoading(true)
      searchItems(combobox.deferredValue).then((res) => {
        combobox.setMatches(res.map((i) => i.value))
        combobox.setLoading(false)
      })
    }
  }, [combobox.deferredValue])

  return (
    <div>
      <ComboboxField id="async-combobox" state={combobox} label="Country" />
      <ComboboxPopover state={combobox} />
    </div>
  )
}

render(<Example />)
```

#### Handling states
<br />

The `loading` and `error` states can be toggled programmatically with `setLoading` and `setError`. Use these functions to interact with your data-fetching library.<br /><br />

```jsx noInline live
const ComboboxField = experimental_ComboboxField
const ComboboxPopover = experimental_ComboboxPopover
const useComboboxState = experimental_useComboboxState

function Example() {
  const combobox = useComboboxState({
    list: ['Brazil', 'Australia', 'France', 'Canada', 'Japan'],
  })

  const toggleLoading = () => {
    combobox.setLoading((l) => !l)
    combobox.setVisible(true)
  }

  const toggleError = () => {
    combobox.setError((e) => !e)
    combobox.setVisible(true)
  }

  return (
    <>
      <Stack direction="row">
        <Button size="small" variant="tertiary" onClick={toggleLoading}>
          toggle loading
        </Button>
        <Button size="small" variant="danger-tertiary" onClick={toggleError}>
          toggle error
        </Button>
      </Stack>
      <ComboboxField id="country-combobox" label="Country" state={combobox} />
      <ComboboxPopover state={combobox} />
    </>
  )
}

render(<Example />)
```

### I18n
<br />

The Combobox component has default translations for the popover strings. Since the label can vary, you should handle this translation. Wrap your app with the [I18nProvider](/experimental/localization#i18nprovider/) to configure the locale.<br /><br />

```jsx noInline live
const ComboboxField = experimental_ComboboxField
const ComboboxPopover = experimental_ComboboxPopover
const useComboboxState = experimental_useComboboxState
const I18nProvider = experimental_I18nProvider

function ComboboxComponent() {
  const combobox = useComboboxState({
    list: ['Brazil', 'Australia', 'France', 'Canada', 'Japan'],
  })

  return (
    <>
      <ComboboxField id="country-combobox" label="Country" state={combobox} />
      <ComboboxPopover state={combobox} />
    </>
  )
}

function RootApp() {
  return (
    <I18nProvider locale="pt-BR">
      <ComboboxComponent />
    </I18nProvider>
  )
}

render(<RootApp />)
```

#### ComboboxState
<br />

The hooks return the ComboboxState, with the values:<br /><br />

| Name            | Type                         | Description                                                                                 |
| --------------- | ---------------------------- | ------------------------------------------------------------------------------------------- |
| value           | `string`                     | Current input value                                                                         |
| setLoading      | `React.SetState<string>`     | Set the value                                                                               |
| deferredValue   | `string`                     | Current input value delayed by a defined `timeoutMs`                                        |
| matches         | `ItemType[]`                 | List of items that were a match for the current input search                                |
| setMatches      | `React.SetState<ItemType[]>` | Set matches value                                                                           |
| selectedItem    | `ItemType`                   | Currently selected item, this props contains the full object, and not just the string value |
| setSelectedItem | `(arg: ItemType) => void`    | Set selected item value                                                                     |
| setLoading      | `React.SetState<boolean>`    | Set loading state                                                                           |
| setError        | `React.SetState<boolean>`    | Set error state                                                                             |
| getOptionValue  | `(item: ItemType) => string` | Same as the one passed to the hook                                                          |
| renderOption    | `(item: ItemType) => string` | Same as the one passed to the hook                                                          |

<details>
  <summary><h4>What should happen when the merchant is typing?</h4></summary>
  <div>
    Merchants type text in the Combobox to perform a search among the available options and select one of the resulting options.
  </div>
  <div>
    <ul>
      <li>Implement a debounce logic so that search results are not loaded prematurely in the combobox while the merchant is still typing.</li>
      <li>While the search is being processed, display the previous results until the new ones are loaded.</li>
      <li>The Popover with the available options should expand or shrink in height depending on the number of loaded results.</li>
      <li>If there are no search results, don't display a popover or display a popover only with the action to create a new option when it exists.</li>
    </ul>
    <br />
    <strong>What to avoid:</strong><br /><br />
    <ul>
      <li>Donâ€™t change the search query to reflect the option that the merchant focused on with the keyboard. Prefer to maintain the search query until the merchant chooses an option or manually clears the field.</li>
      <li>Don't implement a search where the results take more than 1 second to update. Prefer to have a search with better performance so that loading feedback isnâ€™t necessary.</li>
    </ul>
  </div>

  <div>
    Merchants type text in the Combobox to perform a search among the available options and select one of the resulting options.
  </div>
  <div>
    <ul>
      <li>Whenever possible, the loading process should take less than 1 second and the previous results should be displayed until the new ones are loaded.</li>

<details>
  <summary><h4>When should the error state of a form field be visible?</h4></summary>
  <div>
    The validation behavior should communicate the error at the right moment and make it easier to fix. To minimize frustration, follow these recommendations:
  </div>
  <div>
    <ul>
      <li>Show the error in a specific field when the merchant removes its focus or in any fields when they attempt to submit the form. For example, show an error when the merchant removes the focus of an empty mandatory field.</li>
      <li>Remove the error state of the field as soon as the merchant modifies its value.</li>
      <li>When the merchant creates an option that has the same value as an option that is already chosen, mark this option with an error state and add the "<em>Some values are duplicated</em>" error text in the field.</li>
      <li>When the field expects a specific format (e.g. email) and the merchant creates an option that doesnâ€™t conform to this format, show the option in an error state.</li>
      <li>The merchant may type a search query and then remove the focus of the field without choosing one of the search results. If the value matches one of the available options, choose this option for them. If the value doesn't match one of the available options but the field allows new options to be created, create an option with the value. If it doesn't match an option and one can't be created, transform the query into an option in an error state.</li>
    </ul>
    <br />
    <strong>What to avoid:</strong><br /><br />
    <ul>
      <li>Don't display only the error state of the option. Prefer to also display the error state of the field and an error text any time there is an error.</li>
    </ul>
  </div>
</details>

<details>
  <summary><h4>When should a form field be disabled?</h4></summary>
  <div>
