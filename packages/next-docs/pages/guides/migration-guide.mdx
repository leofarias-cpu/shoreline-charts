# Migration Guide

Migrating your Admin projects from `@vtex/admin-ui` to `@vtex/shoreline-components`.

Shoreline is a major update on the design system library with focus to provide a more stable and reliable library, with a better DX and a more consistent UI.

You can notice some breaking changes on the components API, this was necessary to improve the DX and to make the components more reliable. 
We also removed some components that were not being used and some that were not being used correctly.

For upcoming updates you will notice that stability will be our focus and we will work hard to make any breaking changes.

## Getting Started

Upgrading to Shoreline requires several automated and manual migrations of tokens, components, and component prop names that have been removed, replaced, or renamed and also changes on the way the style is applied to the components.
Those upgrades can be done on a component by component basis, since `admin-ui` and `Shoreline` can coexist in the same project.

```sh npm2yarn copy
npm i @vtex/shoreline-components
```

1. Removed the `action` property. <br/>

| `variant="critical"` | `variant="critical"` |

</details>
